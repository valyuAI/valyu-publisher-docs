---
title: Seller Usage
api: POST /analytics/seller-usage
description: Get aggregated usage data as a time series, broken down by data source.
---

<ParamField body="timeScale" type="string" required>
  Time bucket granularity. One of:
  - `h` - Hourly
  - `d` - Daily
  - `w` - Weekly
  - `m` - Monthly
</ParamField>

<ParamField body="period" type="integer" required>
  Number of time buckets to return. For example, `7` with `timeScale: "d"` returns the last 7 days. Maximum: 365.
</ParamField>

<ParamField body="includeCount" type="boolean" default="false">
  When `true`, includes retrieval count fields alongside earnings (e.g., `All_count`, `source/academic_count`).
</ParamField>

## Response fields

The response is an array of time buckets. Each bucket contains:

| Field | Type | Description |
|-------|------|-------------|
| `hour` / `day` / `week` / `month` | string | Timestamp (key matches your `timeScale`) |
| `All` | number | Total earnings across all sources (USD) |
| `source/xxx` | number | Earnings for a specific source |
| `All_count` | number | Total retrieval count (only when `includeCount: true`) |
| `source/xxx_count` | number | Per-source retrieval count (only when `includeCount: true`) |

<Note>
  Source keys are dynamic and depend on which sources serve your content. Common values include `source/academic`, `source/web`, `source/financial`.
</Note>
